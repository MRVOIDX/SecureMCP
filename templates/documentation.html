SecureMCP uses a combination of pattern matching and AI analysis:

  1. Pattern Detection: Scans your code against 81 security patterns
  2. AI Analysis: Groq AI analyzes each finding for context
  3. Recommendations: Provides detailed explanations and fix suggestions
  4. Reporting: Generates comprehensive security reports

üì§ Uploading Projects

SecureMCP accepts ZIP files. To prepare your project:

  1. Compress your project folder
  2. Ensure it's in ZIP format
  3. Maximum size: 50MB
  4. Include all relevant source files

üîç Scanning

The scanner analyzes:

  • Python, JavaScript, TypeScript files
  • Java, PHP, Ruby, Go, C/C++ code
  • HTML, Vue, Svelte templates
  • Configuration files (.env, .yml, .json, etc.)

It checks for hardcoded secrets, SQL injection, XSS, command injection, and 17+ other vulnerabilities.

üìÑ Reports

üí° Fix Suggestions

Each finding includes:

üîë Secret Management

Don't: Hardcode API keys, passwords, or tokens in your source code

Do: Use environment variables or dedicated secret management tools:

  • Python: python-dotenv
  • Node.js: dotenv
  • Cloud: AWS Secrets Manager, Azure Key Vault

üîí HTTPS

Always use HTTPS for production applications. This prevents:

  • Man-in-the-middle attacks
  • Data interception
  • Credential theft

üë§ Authentication

Best practices:

  • Never disable authentication
  • Use strong password hashing (bcrypt, argon2)
  • Implement rate limiting
  • Use secure session management
  • Require HTTPS for auth endpoints
